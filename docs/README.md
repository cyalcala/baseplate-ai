# Baseplate Docs

This directory is the canonical Markdown source for Baseplate's decision product.

Notion can remain the private command center and tracker, but polished decision artifacts should live here so changes are versioned, reviewable, and publishable.

## Folders

- `decisions/` - decision pages such as RAG-001.
- `diffs/` - Architecture Diff posts explaining what changed and why.
- `evidence/` - source queues, reviewed source notes, and evidence schema.
- `offers/` - validation offers and customer-facing templates.
- `playbook/` - paid/free playbook drafts.
- `reference/` - comparison tables and reusable reference material.
- `validation/` - outreach messages, operating plans, and tracking files.

## Update Rule

Do not change a decision just because a new paper or vendor post exists.

Change a decision only when the evidence is stronger, directly relevant, contradictory to the current default, or reveals a major risk.
